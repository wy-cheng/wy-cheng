1. POM

It is an XML file that contains information about the project and configuration details used by Maven to build the project.

POM的主要构成元素：

  • project
  • modelVersion
  • groupId
  • artifactId
  • version
  • name This element indicates the display name used for the project. This is often used in Maven’s generated documentation.
  • url This element indicates where the project’s site can be found. This is often used in Maven’s generated documentation.
  • properties
  • dependencies
  • build

4. Dependency Mechanism

4.1 传递依赖

通过依赖传递，避免了需要手动将所有依赖导入。

  • 依赖调解（Dependency mediation）—— Maven picks the “nearest definition”. You can always guarantee a version by declaring it explicitly in your project’s POM. if two dependency versions are at the same depth in the dependency tree, the first declaration wins.

4.2 Dependency Scope

  • compile
    This is the default scope.

  • provided

    It is not transitive.

  • runtime
    This scope indicates that the dependency is not required for compilation, but is for execution.

  • test
    This scope is not transitive.

  • system
    This scope is similar to provided except that you have to provide the JAR which contains it explicitly. The artifact is always available and is not looked up in a repository.

  • import
    This scope is only supported on a dependency of type pom in the section. It indicates the dependency is to be replaced with the effective list of dependencies in the specified POM's section. Since they are replaced, dependencies with a scope of import do not actually participate in limiting the transitivity of a dependency.
